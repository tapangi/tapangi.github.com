Technology is a tool not a solution

We are technology agnostic. There are dozens of languages, frameworks and libraries. All have their unique strengths and weaknesses. We strive to continually learn new and emerging technologies, while mastering ones we are already familiar with.

Social

These days we can't imagine our world without Facebook and Twitter. Almost everyone on the internet is part of some social network. We help our clients build engaging interactions through sharing, competing and using their social identities to quickly jump in and start using the app.

Mobile

Today's modern applications and services are expected to be available anywhere, including your own pocket. Most common challenge is presence on all important platforms. We crank out native Android and iOS. Sometimes the best approach is to build a hybrid. We got that covered with PhoneGap/Cordova. Sometimes a mobile web app is the right choice. We've done it all.

Cloud

Though web is dead, web applications are and will remain very important. Using HTML5 and related technologies we create build rich, responsive user experiences and tie them with smart RESTful APIs. Cloud computing lowered the barriers and costs to deploy robust and scalable backends in almost any technology. We regularly take advantage of Ruby on Rails, MySQL, Postgres, Redis and jQuery, but are no strangers to Java, PHP, Web Sockets, Node.js and others.

Apps are products, not projects

Building applications is a lot more involving than knocking out closed-end projects. Software products have a life on their own. They are meant to provide value to the users, change, adapt and improve. This should not come with ever increasing maintenance burden and slowdown.
This is why we are big fans of agile methodologies as well as lean startup principles. We found that they go perfectly hand in hand. On top of that they produce results. And we are all about results.

Kick-Off

  • listen
  • ask the right questions
  • identify the target audience
  • identify epics and stories
  • identify scope of MVP

Analyze

  • break down epics into user stories
  • prioritize the stories
  • review any lessons learned from previous iterations
  • identify new learning objectives and metrics
  • decide whether to pivot or persevere (if necessary)

Architect

  • iterate on User Experience (wireframes)
  • iterate on UI and feature design (comps)
  • select/refine/rethink technology stack
  • identify deployment infrastructure an strategy

Build

  • select top priorities for iteration
  • code and unit test
  • perform acceptance testing
  • integrate and test some more

Launch

  • performance testing
  • deployment
  • promotion

Scale

  • analyze metrics
  • identify bottlenecks
  • set up priorities for next iteration
